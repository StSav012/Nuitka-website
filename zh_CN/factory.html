工厂说明书

工厂的基本原理

master 和 develop 两个分支都应该在任何时候为人们工作。对于问题背景下的实验和一般来说尚未准备好的功能，我们有工厂分支。

你可能已经来到这里，因为你被要求检出在 factory 上为你做的改变。

如何获得 factory

git

git clone --branch factory https://github.com/Nuitka/Nuitka.git

pip

pip install -U --force-reinstall "https://github.com/Nuitka/Nuitka/archive/factory.zip"

Word of Warning

这个个人分支可能包括各种愚蠢的错误，例如不能在所有版本中执行，崩溃，根本无法工作。它们也经常在没有通知的情况下改变。因此，请只在手头的问题上使用它，甚至比通常更多，风险自负。

一旦确认并发现好的，修复通常会在相对较短的时间内出现在 develop 分支上，从那时起尽量使用这些分支，并等待那里出现的变化。
