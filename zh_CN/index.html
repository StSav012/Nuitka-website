Nuitka 网站

致谢

请在各方面给予帮助和改进，拼写错误、更酷的技术、更好的外观、更多的信息等，所有这些都是值得赞赏和必要的。

现在，这是非常有可能改善的。轮到你登场！

用法

这是用来建立 Nuitka 网站。各种改进都是受欢迎的。你将需要 python 3.9 和 pipenv 来构建它，到目前为止，这只在 Linux 上完成，但它应该也可以在 Windows 上工作，你可以把它作为你的贡献的一部分。

# Creates a virtualenv with all that is needed to develop the
# site.
python3 -m pip install -U invoke
python3 -m invoke virtualenv

# Now lets build the site, to see if it's all correct.
python3 -m invoke doc

# Optionally build the localized sites.
python3 -m invoke intl -l zh_CN
python3 -m invoke intl -l de_DE
...

# Start local web server with the site, and do automatic
# rebuilds. Localization are rebuilt if they were previously
# built.
python3 -m invoke run -t serve-site

# Browse it (Linux)
xdg-open http://localhost:8080
# Browse it (Windows)
explorer http://localhost:8080

Sphinx 用法

所以这个网站基本上是自动化的，从 Nuitka 的 git 库中导入一些文件，分割开来，例如 Changelog.rst 进入页面，否则使用 Sphinx 渲染它。参考 Sphinx 文档，特别是 Read the Docs 主题和 ABlog 的文档。

翻译

非常欢迎翻译 Nuitka 网站。我们有一个团队可以帮助那些想要处理它的人，他们拥有对存储库的推送权。目前使用的是 translation 分支，但计划是将其合并，并在 main 分支中提供基础设施。

下面是要翻译的文件及其样式的表格。这里没有提到的一切，如果有疑问，都应该询问。并不是所有的页面都适合翻译。

重要

在翻译之前，希望你也能真正地检查一下内容，是否是你认为的有意义，是否它可以有更好的结构。不幸的是，这是非常需要的。

备注

There are plans to get rid of pages and doc sub-folders of the site folder doc. For URLs, these are plain bad, and would e.g. be much better as python-compiler and the like. A plan will be devised and put into place, but it will just move and renamed pages.

目录 posts/

在这个时候，博客帖子，特别是旧的帖子不应该被翻译。我相信，新内容通常会以 post 的形式创建，然后移到页面进行翻译。当前的例子是教程样式的页面，包括截图，这取决于 Python 安装程序在你的语言中的外观，浏览器和 shell 提示符，甚至截图可能需要翻译。不过，这是以后的事情了。

页面 pages/donations.rst

是的，请讲吧。如果需要提示 Paypal，请告诉我，或者当 Paypal 在你们国家无法使用时，我们可以使用哪些替代方式。

页面 pages/overview.rst

请推迟这个。它目前只是 index.rst 的副本并没有链接，所以现在请忽略它。我们的计划是让用户感觉很好，就像一个十字路口的入口页面，这样就会出现主页这样的页面。

Page pages/pyside2.rst

非常重要的一页，我想要更多。Nuitka 将自己从插件链接到这个页面，它是一个登陆页面，告知用户可能出现的问题。我们需要一些样板，以及添加这些的一般方法。我认为我们应该通过这些用户提示页面来帮助用户从真正的 Nuitka 中找到信息，甚至针对他们的需求进行本地化。

页面 pages/support.rst

这篇文章还没有提到 Discord 服务器，我很快会补充的。非常重要的页面，应该从很多地方链接。

页面 pages/gsoc2019.rst

这是历史信息，翻译没有任何意义，不知道该怎么处理它。但如果 Nuitka 再做一次，我们将会在下一次进行理想的翻译。

页面 pages/impressum.rst

这是我国法律要求的，译者可能会想要在这里添加他们的信息。我不能对翻译的内容负责，因为在很多情况下我无法核实。

页面 pages/Presentations.rst

这部电影可能需要更多的爱情内容。这是在建设中。我想浏览一下这个博客并链接所有来自那里的信息。但如果你知道用自己的语言写的东西，请继续添加。

页面 Streaming.rst

由于是英文提供的，不知道是否有道理的，我也会在不久的将来更新它，但明确表示这将是一个英语的内容，我认为这应该翻译在基本层面上的页面，然后翻译内容吗？

页面 doc/api-doc.rst

翻译没有任何意义，而且目前也做不到。API 文档是用 Spinx 生成的，Doxygen 文档是没有爱的。

页面 doc/commercial.rst 和 文件夹 commercial

翻译非常受欢迎。随着时间的推移，会有更多的内容添加进来。关于付款方式，请查看它们是否对你的国家有效，如果不有效，请帮助我找到其他方式。例如，我在中国被拒绝使用支付宝，但也许其他东西可以起作用。比如俄罗斯就没有 Paypal（我的意思是这仍然是一个替代选择）。

页面 doc/factory.rst

翻译出来非常有用。

页面 doc/welcome.rst

这对我来说只是一个游乐场，不要翻译，除非我们想玩翻译机制。例如，我们希望在某个时候拥有一个智能语言切换器，并可以尝试它。

页面 doc/Changelog.rst

不要翻译，有着奇怪的工作量。

页面 download.rst

非常欢迎，但要小心 download.rst.j2 才是真正的来源。表格会生成到文档中，这可能在技术上是一个比较困难的情况，所以先不要急着做，直到问题解决。

页面 developer-manual.rst

翻译没有任何意义。但其中可能有属于用户手册的内容，或者应该被拆分出来。

页面 roadmap.rst

很像更新日志，没有那么多工作，但也没有那么重要。

页面 doc/user-manual.rst

这一点在我看来是最重要的，优化部分需要我的认真更新，也许忽略它，直到我到达那里。复习在这里很适用。我认为教程部分可能会被分解成独立的文档。

图片优化

# Optimize PNG files like this, normally not needed, this
# is lossless.
sudo apt-get install optipng
find . -iname *.png -a -type f -exec optipng -o7 -zm1-9 {} \;

# Optimize JPEG files like this, normally not needed, this
# is lossless.
sudo apt-get install jpegoptim
find . -iname *.jpg -a -type f -exec jpegoptim {} \;
